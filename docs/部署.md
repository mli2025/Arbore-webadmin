# Web Admin 宿主机部署说明

在 **Ubuntu 服务器**上一次性完成部署：后端 API 宿主机运行（端口 8002），前端由 Nginx 提供（端口 9000）。

**推荐**：使用 `web-admin.tar.gz` 包部署（方式一）；无包时用分步上传（方式二）。

---

## 打包 web-admin.tar.gz（本机执行，用于发布）

在项目根目录执行，生成 `init/web-admin.tar.gz`：

```powershell
cd E:\Arbore\web-admin
tar -czf init\web-admin.tar.gz -C init web-admin
```

生成的 `init/web-admin.tar.gz` 可放至公用网络供新服务器下载。

---

## 约定

| 项 | 值 |
|----|-----|
| 目标路径 | `/home/arbore/console/web-admin` |
| 后端端口 | 8002（uvicorn） |
| 前端端口 | 9000（Nginx） |
| 系统用户 | `arbore` |
| 本机项目路径 | 以 `E:\Arbore\web-admin` 为例，按你实际路径替换 |

## 方式一：使用 web-admin.tar.gz 部署（推荐）

### 第一步：服务器环境准备（新机执行一次）

同「方式二」第一步（adduser、apt nginx/python、Docker、Docker Compose V2）。

### 第二步：上传并解压 web-admin.tar.gz

将 `web-admin.tar.gz` 上传到服务器后，**在服务器**执行：

```bash
mkdir -p /home/arbore/console
# 假设 web-admin.tar.gz 已上传到 /home/arbore/ 或当前目录
tar -xzf web-admin.tar.gz -C /home/arbore/console/
```

解压后目录结构：`/home/arbore/console/web-admin/`（含 backend、frontend/dist、deploy、docker-compose.env 等）。

### 第三步：部署后端（venv + systemd）

```bash
cd /home/arbore/console/web-admin/backend
rm -rf venv
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
pip install uvicorn[standard]
deactivate
```

```bash
sudo cp /home/arbore/console/web-admin/deploy/arbore-web-admin-api.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable arbore-web-admin-api
sudo systemctl restart arbore-web-admin-api
```

### 第四步：Docker 权限与 Nginx

```bash
sudo usermod -aG docker arbore
sudo systemctl restart docker
```

```bash
sudo chmod o+x /home/arbore /home/arbore/console /home/arbore/console/web-admin /home/arbore/console/web-admin/frontend /home/arbore/console/web-admin/frontend/dist
sudo chown -R arbore:www-data /home/arbore/console/web-admin/frontend/dist
sudo chmod -R 755 /home/arbore/console/web-admin/frontend/dist
```

```bash
sudo cp /home/arbore/console/web-admin/deploy/web-admin.conf /etc/nginx/conf.d/
sudo sed -i 's/server_name localhost;/server_name localhost 你的服务器IP;/' /etc/nginx/conf.d/web-admin.conf
sudo nginx -t
sudo systemctl reload nginx
```

### 第五步及以后

从「第六步：部署 arbore-flow」起，与方式二相同。

## 方式二：分步上传部署

### 第一步：服务器环境准备（新机执行一次）

在服务器上**逐条**执行（建议用 root 或带 sudo 的用户）：

```bash
sudo adduser arbore
sudo usermod -aG sudo arbore
sudo mkdir -p /home/arbore/console
sudo chown -R arbore:arbore /home/arbore/console
```

```bash
sudo apt-get update
sudo apt-get install -y nginx python3-venv python3-pip
```

（若该服务器还需跑 Docker 等其他服务，按下面安装 Docker，并安装 Docker Compose V2。）

安装 Docker：

```bash
sudo apt-get install -y docker.io
sudo systemctl enable --now docker
```

安装 Docker Compose V2（若 `apt-get install docker-compose-plugin` 找不到包，则用下面方式直接安装 CLI 插件二进制）：

```bash
mkdir -p ~/.docker/cli-plugins
COMPOSE_VERSION="v2.39.2"
curl -L "https://github.com/docker/compose/releases/download/${COMPOSE_VERSION}/docker-compose-linux-x86_64" -o ~/.docker/cli-plugins/docker-compose
chmod +x ~/.docker/cli-plugins/docker-compose
docker compose version
```

如果服务器无法访问 GitHub（下载很慢/失败），可用 **本机 WSL（已安装 Docker Desktop）** 复制出 Linux 版 compose 插件再上传：

1) 在本机 WSL 执行，找到插件路径（任选一个存在的目录）：

```bash
ls -la /usr/libexec/docker/cli-plugins/ 2>/dev/null
ls -la /usr/lib/docker/cli-plugins/ 2>/dev/null
ls -la /usr/local/lib/docker/cli-plugins/ 2>/dev/null
```

通常会看到 `docker-compose` 文件。把它复制到 Windows 盘（示例放到 E 盘）。
若它是符号链接，建议用 `cp -aL` 复制为独立文件：

```bash
cp -aL /usr/local/lib/docker/cli-plugins/docker-compose /mnt/e/Arbore/web-admin/docker-compose
```

2) 在本机 PowerShell 上传到服务器：

```powershell
scp E:\Arbore\web-admin\docker-compose arbore@119.45.131.123:/home/arbore/
```

3) 在服务器安装到 CLI 插件目录并验证：

```bash
mkdir -p ~/.docker/cli-plugins
mv /home/arbore/docker-compose ~/.docker/cli-plugins/docker-compose
chmod +x ~/.docker/cli-plugins/docker-compose
docker compose version
```

## 第三步：服务器部署后端（venv + systemd）

在**服务器**上执行：

```bash
cd /home/arbore/console/web-admin/backend

source venv/bin/activate
pip install -r requirements.txt
pip install uvicorn[standard]

```

上传并安装 systemd 单元。**本机**执行：

```powershell
scp E:\Arbore\web-admin\deploy\arbore-web-admin-api.service arbore@119.45.131.123:/home/arbore/
```

**服务器**执行：

```bash
sudo cp /home/arbore/arbore-web-admin-api.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable arbore-web-admin-api
sudo systemctl restart arbore-web-admin-api
sudo systemctl status arbore-web-admin-api --no-pager -l
```

验证 8002：

```bash
curl -s -o- -w "\nHTTP %{http_code}\n" http://127.0.0.1:8002/api/v1/version
```

应返回 JSON 且 `HTTP 200`。

### 3.1 让 Web Admin 能检测 Docker 服务（必须做）

如果你在版本接口里看到：
- `docker_client_available: false`
- `docker_api_available: false`

说明后端 **无法访问本机 Docker**，会导致页面“检测不到其它服务”。按下面做一次即可。

在**服务器**上逐条执行：

```bash
sudo systemctl enable --now docker
sudo usermod -aG docker arbore
sudo systemctl restart docker
sudo systemctl restart arbore-web-admin-api
```

注意：执行 `sudo usermod -aG docker arbore` 后，需要**重新登录一次 SSH**（或在当前会话执行 `newgrp docker`）让组权限生效，否则会出现 `permission denied while trying to connect to the Docker daemon socket`。

重新验证（应至少有一项变为 true）：

```bash
curl -s -o- -w "\nHTTP %{http_code}\n" http://127.0.0.1:8002/api/v1/version
```

## 第六步：部署 arbore-flow（端口 5678，docker run）

本步骤对应 `flow構建方法.md`。你已将镜像 tar 上传到服务器（示例文件：`/home/arbore/console/arbore-prod.tar`）。

> **数据库路径固定规则（重要）**：一律以老业务库为准：
> `/home/arbore/console/data/arbore-flow/database.sqlite`
>
> 启动 `arbore-flow` 时必须同时设置：
> - `DB_TYPE=sqlite`
> - `DB_SQLITE_DATABASE=/app/data/arbore-flow/database.sqlite`
> - `N8N_DATABASE_SQLITE_DATABASE=/app/data/arbore-flow/database.sqlite`
> - `N8N_DB_SQLITE_DATABASE=/app/data/arbore-flow/database.sqlite`
>
> 否则可能误读默认库 `/app/data/.n8n/database.sqlite`，出现 `/setup` 初始化页面。

### 6.1 加载镜像（服务器执行）

```bash
cd /home/arbore/console
docker load -i arbore-prod.tar
```

### 6.2 准备数据目录（服务器执行，首次部署）

```bash
mkdir -p /home/arbore/console/data
mkdir -p /home/arbore/console/custom-nodes
touch /home/arbore/console/license.json
```

### 6.3 启动/重启容器（服务器执行）

注意：镜像名需要与你导入的标签一致。指南示例是 `arbore-prod:001.0063`；如果你实际标签不同，请把下面命令里的镜像名替换为你的实际值。

```bash
docker stop arbore-flow 2>/dev/null
docker rm arbore-flow 2>/dev/null
docker run -d \
  --name arbore-flow \
  -p 5678:5678 \
  -v /home/arbore/console/data:/app/data \
  -v /home/arbore/console/custom-nodes:/app/custom-nodes \
  -v /home/arbore/console/license.json:/app/license.json \
  --restart unless-stopped \
  --entrypoint "" \
  -e N8N_DEFAULT_LOCALE=en \
  -e DB_TYPE=sqlite \
  -e DB_SQLITE_DATABASE=/app/data/arbore-flow/database.sqlite \
  -e N8N_DATABASE_SQLITE_DATABASE=/app/data/arbore-flow/database.sqlite \
  -e N8N_DB_SQLITE_DATABASE=/app/data/arbore-flow/database.sqlite \
  arbore-prod:001.0063 \
  /bin/sh /app/start-prod.sh
```

查看日志：

```bash
docker logs -f arbore-flow
```

浏览器访问：`http://<服务器IP>:5678`

### 6.4（可选）让 arbore-flow 访问 arbore-func 的 PostgreSQL

如果你在 flow（n8n）里使用 PostgreSQL 节点，希望直接访问本机 `arbore-func 2.x` 的数据库容器 `arbore-postgres-nocobase`，需要让两个容器在同一 Docker 网络中。

`arbore-func-v2` 默认使用 `docker compose` 创建网络（通常名为 `arbore-func-v2_arbore-2x`）。你可以：

1. **给已运行的 arbore-flow 挂上该网络（临时/补救）**：

```bash
docker network ls
# 找到名字类似 arbore-func-v2_arbore-2x 的网络
docker network connect arbore-func-v2_arbore-2x arbore-flow
```

然后在 flow 的 PostgreSQL 节点中使用（**库名必须与 `.env` 中 `POSTGRES_NOCOBASE_DB` 一致**，按本文 7.3 默认为 `nocobase2x`，不要填成 `arbore`）：

- Host：`arbore-postgres-nocobase`
- Port：`5432`
- Database：`nocobase2x`（若你改过 `POSTGRES_NOCOBASE_DB`，则填该值）
- User / Password：与 `.env` 中 `POSTGRES_NOCOBASE_USER`、`POSTGRES_NOCOBASE_PASSWORD` 一致

若界面仍报 “Couldn't connect”，在服务器上自检：

```bash
docker inspect arbore-flow --format '{{json .NetworkSettings.Networks}}'
# 输出中应包含 arbore-func-v2_arbore-2x

docker exec arbore-postgres-nocobase pg_isready -U arbore
```

2. **以后重启时，让 arbore-flow 默认加入该网络（推荐正式命令）**：

```bash
docker stop arbore-flow 2>/dev/null
docker rm arbore-flow 2>/dev/null

docker run -d \
  --name arbore-flow \
  --network arbore-func-v2_arbore-2x \
  -p 5678:5678 \
  -v /home/arbore/console/data:/app/data \
  -v /home/arbore/console/custom-nodes:/app/custom-nodes \
  -v /home/arbore/console/license.json:/app/license.json \
  --restart unless-stopped \
  --entrypoint "" \
  -e N8N_DEFAULT_LOCALE=en \
  -e DB_TYPE=sqlite \
  -e DB_SQLITE_DATABASE=/app/data/arbore-flow/database.sqlite \
  -e N8N_DATABASE_SQLITE_DATABASE=/app/data/arbore-flow/database.sqlite \
  -e N8N_DB_SQLITE_DATABASE=/app/data/arbore-flow/database.sqlite \
  arbore-prod:001.0063 \
  /bin/sh /app/start-prod.sh
```

这样，新服务器只要按上述顺序部署 arbore-func-v2 和 arbore-flow，并执行网络连接步骤，flow 中的 PostgreSQL 节点即可直接访问 `arbore-postgres-nocobase`。

## 第七步：部署 arbore-func 2.x（docker compose）

本步骤对应 `arbore-func-2x-新服务器部署指南.md`。你已将镜像 tar 上传到服务器（示例文件：`/home/arbore/console/arbore-func-v2-images.tar`，以及可能用到的 `postgres-16.tar`、`python-3.11-slim.tar`）。

### 7.1 创建部署目录（服务器执行）

```bash
mkdir -p /home/arbore/arbore-func-v2
mkdir -p /home/arbore/arbore-func-v2/data/arbore-func-2x/storage
mkdir -p /home/arbore/arbore-func-v2/data/postgres-2x
```

### 7.2 加载镜像（服务器执行）

```bash
docker load -i /home/arbore/console/arbore-func-v2-images.tar
```

如果你也上传了 Postgres / Python 基础镜像（可选），也可以一起导入：

```bash
docker load -i /home/arbore/console/postgres-16.tar
docker load -i /home/arbore/console/python-3.11-slim.tar
```

导入后查看镜像名（用于下一步写 compose 的 `image:`）：

```bash
docker images | grep -E "nocobase|arbore-func|postgres|python"
```

### 7.3 写入环境变量文件（服务器执行）

```bash
cat > /home/arbore/arbore-func-v2/.env << 'EOF'
POSTGRES_NOCOBASE_DB=nocobase2x
POSTGRES_NOCOBASE_USER=arbore
POSTGRES_NOCOBASE_PASSWORD=arbore
ARBORE_FUNC_APP_KEY=arbore-secret-key-2024
ARBORE_FUNC_ENCRYPTION_KEY=arbore-secret-key-2024
EOF
```

### 7.4 创建 docker-compose.yml（服务器执行）

把 `image:` 改成你在 7.2/7.2 的 `docker images` 里看到的实际镜像名（例如 `nocobase/nocobase:beta-full` 或你导入出来的 `arbore-func-2x:latest`）。

```bash
cat > /home/arbore/arbore-func-v2/docker-compose.yml << 'EOF'
services:
  postgres-2x:
    image: postgres:16
    # 为了让 Web Admin 能识别数据库容器，建议命名为 arbore-postgres-nocobase
    container_name: arbore-postgres-nocobase
    restart: unless-stopped
    environment:
      POSTGRES_DB: ${POSTGRES_NOCOBASE_DB:-nocobase2x}
      POSTGRES_USER: ${POSTGRES_NOCOBASE_USER:-arbore}
      POSTGRES_PASSWORD: ${POSTGRES_NOCOBASE_PASSWORD:-arbore}
      PGDATA: /var/lib/postgresql/data/pgdata
    volumes:
      - ./data/postgres-2x:/var/lib/postgresql/data
    networks:
      - arbore-2x
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_NOCOBASE_USER:-arbore}"]
      interval: 10s
      timeout: 5s
      retries: 5

  arbore-func-2x:
    image: nocobase/nocobase:beta-full
    # 为了让 Web Admin 能识别应用容器，建议命名为 arbore-func
    container_name: arbore-func
    restart: unless-stopped
    environment:
      APP_KEY: ${ARBORE_FUNC_APP_KEY:-arbore-secret-key-2024}
      ENCRYPTION_FIELD_KEY: ${ARBORE_FUNC_ENCRYPTION_KEY:-arbore-secret-key-2024}
      DB_DIALECT: postgres
      DB_HOST: postgres-2x
      DB_PORT: 5432
      DB_DATABASE: ${POSTGRES_NOCOBASE_DB:-nocobase2x}
      DB_USER: ${POSTGRES_NOCOBASE_USER:-arbore}
      DB_PASSWORD: ${POSTGRES_NOCOBASE_PASSWORD:-arbore}
      TZ: Asia/Shanghai
    volumes:
      - ./data/arbore-func-2x/storage:/app/nocobase/storage
    ports:
      # Web Admin 默认按 13000 展示 arbore-func
      - "13000:80"
    networks:
      - arbore-2x
    depends_on:
      postgres-2x:
        condition: service_healthy

networks:
  arbore-2x:
    driver: bridge
EOF
```

### 7.5 启动（服务器执行）

```bash
cd /home/arbore/arbore-func-v2
docker compose up -d
docker compose ps
```

查看日志：

```bash
cd /home/arbore/arbore-func-v2
docker compose logs -f arbore-func-2x
```

浏览器访问：`http://<服务器IP>:13000`

## 第八步：验证

- 浏览器访问：`http://<服务器IP>:9000`，应打开 Web Admin 界面。
- 若需许可证：按下面步骤注册（否则页面会显示 `LICENSE_NOT_FOUND`，并限制启动/重启等操作）。

### 8.1 获取主机标识（Host ID）

你可以在页面的许可证页看到 Host ID（格式类似 `host-00afa07e93fe50eb`），也可以在服务器上查询：

```bash
curl -s http://127.0.0.1:8002/api/v1/license/hardware-fingerprint
```

### 8.2 在本机生成 Validation Code

在本机 PowerShell 执行（需要已安装 Node.js；`<companyId>`、`<companyName>` 用你的企业信息替换）：

```powershell
cd E:\Arbore\web-admin\validation
node .\validation-code-generator.js "<companyId>" "host-00afa07e93fe50eb"
```

输出里的 `Validation Code` 即为要填入的校验码。

### 8.3 注册许可证

方式 A：在 Web 页面填写并提交（推荐）：Company ID / Company Name / Validation Code。

方式 B：在服务器用 API 注册（Form 表单）：

```bash
curl -X POST http://127.0.0.1:8002/api/v1/license/register \
  -F "companyId=<companyId>" \
  -F "companyName=<companyName>" \
  -F "validationCode=<validationCode>"
```

注册后验证：

```bash
curl -s http://127.0.0.1:8002/api/v1/license/validate
```

许可证会写入项目的 `config/license.json`（位于后端项目根目录的 `config/` 下）。

## 部署后必做：修复 systemd PATH（自定义服务依赖）

默认的 `arbore-web-admin-api.service` 中 `PATH` 只包含 venv 路径，后端调用 `docker compose` 时会报 `No such file or directory: 'docker'`，导致**自定义服务上传后无法自动启动容器**。

部署完成后，在**服务器**上执行一次：

```bash
sudo sed -i 's|Environment="PATH=/home/arbore/console/web-admin/backend/venv/bin"|Environment="PATH=/home/arbore/console/web-admin/backend/venv/bin:/usr/bin:/usr/local/bin:/usr/sbin:/home/arbore/.docker/cli-plugins"|' /etc/systemd/system/arbore-web-admin-api.service
sudo systemctl daemon-reload
sudo systemctl restart arbore-web-admin-api
```

验证：

```bash
sudo systemctl cat arbore-web-admin-api | grep PATH
```

应看到一行包含 `venv/bin` **和** `/usr/bin` 等系统路径的 `Environment="PATH=..."` 。

## 自定义服务 Docker 镜像要求

通过 Web Admin 界面上传的自定义服务 Docker 镜像，需满足以下条件：

| 要求 | 说明 |
|------|------|
| 宿主机端口 | 必须在 **7000-7999** 范围内 |
| `/health` 端点 | 容器必须在 `http://localhost:{容器内端口}/health` 返回 HTTP 200，否则健康检查失败 |
| 容器内端口 | 上传时需正确填写（如 nginx 默认是 80，不是 7001） |

**示例**：基于 `nginx:alpine` 的最小测试镜像，需要自定义 nginx 配置添加 `/health`：

```nginx
server {
    listen 80;
    location /health {
        return 200 '{"status":"ok"}';
        add_header Content-Type application/json;
    }
    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ =404;
    }
}
```

上传时填写：外部端口 `7001`，容器内端口 `80`。

## HTTPS 配置（9000 / 5678 / 13000）

使用野卡证书 `*.fungo.co.th` 为上述三个端口启用 HTTPS。**必须使用 fungo.co.th 域名访问**，否则浏览器会报证书域名不匹配。

### 第一步：服务器创建证书目录

**在服务器上**执行：

```bash
mkdir -p /home/arbore/console/ssl
```

### 第二步：本机上传三个文件

**在本机 Windows PowerShell** 中执行（`<服务器IP>` 换成实际 IP，如 `119.45.131.123`）：

```powershell
cd E:\Arbore\web-admin
```

| 文件 | 上传到服务器路径 | 本机执行命令 |
|------|------------------|--------------|
| 私钥 | `/home/arbore/console/ssl/privkey.pem` | `scp temp\pkwildcardfungocothcom3y.key arbore@<服务器IP>:/home/arbore/console/ssl/privkey.pem` |
| 证书链 | `/home/arbore/console/ssl/fullchain.pem` | `scp deploy\ssl\fullchain.pem arbore@<服务器IP>:/home/arbore/console/ssl/` |
| Nginx 配置 | `/home/arbore/web-admin-ssl.conf` | `scp deploy\web-admin-ssl.conf arbore@<服务器IP>:/home/arbore/` |

逐条执行：

```powershell
scp temp\pkwildcardfungocothcom3y.key arbore@<服务器IP>:/home/arbore/console/ssl/privkey.pem
scp deploy\ssl\fullchain.pem arbore@<服务器IP>:/home/arbore/console/ssl/
scp deploy\web-admin-ssl.conf arbore@<服务器IP>:/home/arbore/
```

### 第三步：修改 Docker 端口映射（在服务器上执行）

**3.1 arbore-flow**：改用内部端口 15678，供 Nginx 在 5678 上代理 HTTPS：

```bash
docker stop arbore-flow 2>/dev/null
docker rm arbore-flow 2>/dev/null
docker run -d \
  --name arbore-flow \
  -p 127.0.0.1:15678:5678 \
  -v /home/arbore/console/data:/app/data \
  -v /home/arbore/console/custom-nodes:/app/custom-nodes \
  -v /home/arbore/console/license.json:/app/license.json \
  --restart unless-stopped \
  --entrypoint "" \
  -e N8N_DEFAULT_LOCALE=en \
  -e DB_TYPE=sqlite \
  -e DB_SQLITE_DATABASE=/app/data/arbore-flow/database.sqlite \
  -e N8N_DATABASE_SQLITE_DATABASE=/app/data/arbore-flow/database.sqlite \
  -e N8N_DB_SQLITE_DATABASE=/app/data/arbore-flow/database.sqlite \
  arbore-prod:001.0063 \
  /bin/sh /app/start-prod.sh
```

（镜像名 `arbore-prod:001.0063` 按你实际导入的标签替换。）

**3.2 arbore-func**：编辑 `docker-compose.yml`，把 `"13000:80"` 改成 `"127.0.0.1:13080:80"`，再重启：

```bash
# 用 sed 修改，或手动编辑 /home/arbore/arbore-func-v2/docker-compose.yml
sed -i 's/"13000:80"/"127.0.0.1:13080:80"/' /home/arbore/arbore-func-v2/docker-compose.yml
cd /home/arbore/arbore-func-v2
docker compose up -d
```

### 第四步：启用 Nginx SSL 配置（在服务器上执行）

```bash
# 复制 SSL 配置到 Nginx
sudo cp /home/arbore/web-admin-ssl.conf /etc/nginx/conf.d/
# 删除原 HTTP 配置（避免端口冲突）
sudo rm -f /etc/nginx/conf.d/web-admin.conf
# 检查配置
sudo nginx -t
# 重载 Nginx
sudo systemctl reload nginx
```
