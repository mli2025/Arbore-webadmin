# Web Admin 更新方法

## 文档用途（给助手读，不是给操作者背全文）

- **本文是 OTA 更新流程的唯一规范**：步骤、顺序、命令以本文为准。
- 用户在 Cursor 里要求「一步步指导」时，**助手应先阅读本文**，按 **「复制粘贴流程」** 从阶段 A → B → C 往下带，**不要另起一套说法或临时发挥**。
- 用户只需跟着助手给出的「当前一步」执行；**不要求用户自己通读本文**，但本文需保持与仓库同步，避免助手每次从零推断。

## 方式一：OTA 自动更新（推荐）

利用内置的远程更新功能，通过 Web 界面一键完成后端 + 前端更新，无需手动 scp 和重启。

### 先回答你问的两件事

| 方式 | 要不要 |
|------|--------|
| **git** | **OTA 这条链路不需要。** 你在自己电脑上照常 git 拉代码、改代码；和公司更新服务器之间**不传 git 仓库**，只传打好的 `backend.tar.gz` 等文件。 |
| **cp** | **不需要你在公司服务器上 `cp` 源码。** 本机用 `tar` 打好包，用 **`scp` 上传到** `/tmp/arbore-update/`；目标机上的覆盖由 **OTA 点「立即更新」** 时程序自动完成。 |

**代码怎么「到公司服务器」？** 只有一条路：**本机 PowerShell 里 `scp`**（下面整段命令已按顺序写好，你只需改 IP）。

### 复制粘贴流程（从本机到开 HTTP，按行执行）

**约定：** 下面所有命令假设项目在 `E:\Arbore\web-admin`，公司服务器地址为 `119.45.212.137`。  
**你只需要：** 把 `119.45.212.137` 全文换成你们真实 IP。

**版本号规则（以代码为准，不要手写乱填）：**

- `version.json` 里的 **`"version"` 必须与本次打进 `backend.tar.gz` 的 `backend/main.py` 中 `API_VERSION` 完全一致**（否则更新完成后界面仍会一直提示更新）。
- 下面用脚本**从 `backend/main.py` 自动读取 `API_VERSION`** 生成 `version.json`，无需你填版本号。
- 目标机是否出现「有新版本」：由后端逻辑比较 **远程 `version.json` 的版本** 与 **目标机当前运行的 `API_VERSION`**，仅当远程**更高**时才会提示（版本是否 bump 由你们在改代码时决定，与本文无关）。

#### 阶段 A：只在 Windows PowerShell 里执行

打开 **PowerShell**，**一行一行**粘贴执行（不要合并成一行）：

```powershell
$COMPANY_IP = "119.45.212.137"
```

```powershell
cd E:\Arbore\web-admin
```

```powershell
tar -czf backend.tar.gz backend
```

根据 **`backend/main.py` 里的 `API_VERSION`** 自动生成 `version.json`（不要手改版本号，除非你先改了 `main.py`）：

```powershell
$mainPy = Get-Content "E:\Arbore\web-admin\backend\main.py" -Raw -Encoding UTF8
if ($mainPy -notmatch 'API_VERSION\s*=\s*"([^"]+)"') { throw "未在 backend/main.py 中找到 API_VERSION" }
$apiVer = $Matches[1]
$obj = [PSCustomObject]@{
  version    = $apiVer
  build_time = (Get-Date -Format "yyyy-MM-dd")
  changes    = @("OTA package; API_VERSION=$apiVer in backend/main.py")
}
$jsonText = $obj | ConvertTo-Json
[System.IO.File]::WriteAllText("E:\Arbore\web-admin\version.json", $jsonText, [System.Text.UTF8Encoding]::new($false))
```

构建前端并打包（**每次 OTA 均包含前后端**，不得省略）：

```powershell
cd E:\Arbore\web-admin\frontend
```

```powershell
npm run build
```

```powershell
cd E:\Arbore\web-admin
```

```powershell
tar -czf frontend-dist.tar.gz -C E:\Arbore\web-admin\frontend dist
```

在公司服务器上建好目录（用 ssh，**仍然在本机 PowerShell 执行**）：

```powershell
ssh root@$COMPANY_IP "mkdir -p /tmp/arbore-update"
```

上传后端包、版本文件与前端包（**这就是「把代码传上去」**，传的是压缩包不是裸源码）：

```powershell
scp E:\Arbore\web-admin\backend.tar.gz E:\Arbore\web-admin\version.json E:\Arbore\web-admin\frontend-dist.tar.gz root@${COMPANY_IP}:/tmp/arbore-update/
```

#### 阶段 B：在公司 Linux 服务器上执行（SSH 登录后）

用 `ssh root@119.45.212.137`（IP 换成你的）登录，然后：

```bash
cd /tmp/arbore-update
```

```bash
ls -la
```

应能看到 `version.json`、`backend.tar.gz`、`frontend-dist.tar.gz`。

```bash
python3 -m http.server 9999
```

**保持这个窗口不关**，更新测完再在窗口里按 `Ctrl+C` 结束。

#### 阶段 C：在目标机浏览器里操作

1. 打开管理端，例如 `http://<目标机IP>:9000` → **系统配置**。  
2. **更新服务器地址**填：`http://119.45.212.137:9999`（IP、端口与上面 `http.server` **必须一致**）。  
3. **保存配置** → **立即检查** → 按提示 **立即更新**。

**常见错误：** 填成 `http://IP/` 或 `http://IP`（缺 **`:9999`**）会访问目标机 **80 端口**，拉到的不是 `/tmp/arbore-update` 里的 `version.json`，界面会误报「已是最新版本」或版本号不对。**必须带临时 HTTP 的端口**，例如 `http://119.45.212.137:9999`。

**「启用自动检测」开关：** 仅控制**后台是否每小时自动**请求远程；**手动点「立即检查」**只要填了正确的更新地址即会拉取 `version.json`（与开关无关；若你仍在使用旧版后端，可把开关打开再试一次）。

### 原理

后端启动后会定期检查 `update_url` 指向的 HTTP 服务器，比较 `version.json` 中的版本号与本地 `API_VERSION`。发现新版本后，界面顶部弹出横幅，用户点击"查看更新"→"立即更新"即可自动下载、覆盖、重启。

更新服务器需提供以下文件：

| 文件 | 用途 |
|------|------|
| `version.json` | 版本信息，后端据此判断是否有新版本 |
| `backend.tar.gz` | 后端源码包，解压后覆盖 `.py` 等文件（不覆盖 venv/config/logs） |
| `frontend-dist.tar.gz` | 前端构建产物（OTA 标准流程必带，与后端一并上传） |

### 环境说明（必读，避免第一步就错）

| 你在哪台机器 | 能用的路径 / 命令 |
|--------------|-------------------|
| **Windows 开发机（本机）** | 盘符路径如 `E:\Arbore\web-admin`。可用 `tar`（Win10+ 自带）、`scp`。**没有** `/tmp`，也**不要**执行 `cd /tmp/...`（PowerShell 会当成 `E:\tmp\...` 并报错）。 |
| **公司 Linux 服务器（SSH 进去之后）** | 才有 `/tmp/arbore-update`、`mkdir -p`、`python3 -m http.server`。文档里凡是以 ````bash` 开头的块，**只在这里执行**。 |

**backend.tar.gz 结构要求（与 OTA 解压逻辑一致）：** 压缩包顶层必须是**一个名为 `backend` 的目录**（里面是 `main.py`、`requirements.txt` 等）。在 Windows 项目根目录打包时，用下面「推荐」命令即可满足。

### 完整操作步骤（与上文「复制粘贴流程」同一件事，此处为分步说明）

若你只想跟一条线走，**只用上面「复制粘贴流程」即可**，不必再看下面重复细节。

以"公司服务器 `root@119.45.212.137` 做中转更新服务器，测试机 `arbore@192.168.99.226` 做目标机"为例。

#### 1. 在 Windows 本机准备文件并上传到公司服务器

**推荐（整包后端，最少踩坑）：** 在 **PowerShell** 中逐行执行（路径按你本机修改）：

```powershell
cd E:\Arbore\web-admin
```

```powershell
tar -czf backend.tar.gz backend
```

确认当前目录下新生成了 `backend.tar.gz`（不要先 `cd` 到不存在的 `E:\tmp`）。

在 Windows 上生成 `version.json`：**`"version"` 必须与 `backend/main.py` 中 `API_VERSION` 一致**，推荐用阶段 A 里的 PowerShell 从 `main.py` 自动生成，不要手写版本号。

构建前端并打包（与阶段 A 一致，**每次均执行**）：

```powershell
cd E:\Arbore\web-admin\frontend
npm run build
```

```powershell
cd E:\Arbore\web-admin
tar -czf frontend-dist.tar.gz -C E:\Arbore\web-admin\frontend dist
```

上传到公司服务器（**把 IP 换成你们公司服务器**）。**首次使用**请先在服务器上建好目录（SSH 登录后执行一次即可）：`mkdir -p /tmp/arbore-update`。

```powershell
cd E:\Arbore\web-admin
```

```powershell
scp .\backend.tar.gz .\version.json .\frontend-dist.tar.gz root@119.45.212.137:/tmp/arbore-update/
```

**备选（只改少量后端文件时）：** 也可只传改动的 `.py` 到公司服务器再在那边打 `backend.tar.gz`，见下方「仅在 Linux 上打包」。

#### 2. 在公司 Linux 服务器上准备目录并启动 HTTP（SSH 登录后执行）

首次使用先建目录：

```bash
mkdir -p /tmp/arbore-update
```

若 **第 1 步已用本机打好 `backend.tar.gz` 并 scp 上去**，则**无需**再执行 `tar czf backend.tar.gz`，直接进入目录启动 HTTP 即可。

若 **只在服务器上放了 `backend/main.py` 等散文件**，才在服务器上打包：

```bash
cd /tmp/arbore-update
tar czf backend.tar.gz backend/
```

若服务器上还没有 `version.json`，须在 Windows 按上文从 `backend/main.py` 生成后 `scp` 上去；**勿在服务器上随便写一个与 `main.py` 不一致的版本号**。

启动临时 HTTP 服务：

```bash
cd /tmp/arbore-update
python3 -m http.server 9999
```

> 确认 `/tmp/arbore-update` 下有：`version.json`、`backend.tar.gz`、`frontend-dist.tar.gz`。

#### 3. 在目标机 Web 界面配置更新地址

打开管理界面（如 `http://192.168.18.12:9000`），进入 **系统配置** 标签页：

1. **更新服务器地址**填：`http://119.45.212.137:9999`
2. 点击 **保存配置**
3. 点击 **立即检查** → 应显示"有新版本: vX.X.X"

#### 4. 应用更新

页面顶部会出现橙色横幅"发现新版本"，点击 **查看更新** → **立即更新**。

> **如果横幅按钮无法点击**（CSS 层级问题），在浏览器 F12 控制台执行：
> ```javascript
> fetch('/admin-api/api/v1/update/apply', {method:'POST'}).then(r=>r.json()).then(d=>console.log(d))
> ```

等待 10~30 秒服务自动重启，刷新页面确认版本号更新。

#### 5. 收尾

- 在公司服务器上 **Ctrl+C** 关掉临时 HTTP 服务
- 如果版本号未更新，说明 `main.py` 里的 `API_VERSION` 没改，在目标机上修复：

```bash
sed -i 's/API_VERSION = "旧版本"/API_VERSION = "新版本"/' /home/arbore/console/web-admin/backend/main.py
sudo systemctl restart arbore-web-admin-api
```

### 注意事项

- `version.json` 中的 `"version"` 必须与本次包内 `backend/main.py` 的 `API_VERSION` 一致（推荐用本文阶段 A 脚本从 `main.py` 生成），否则更新后横幅会持续弹出
- OTA 更新**不会覆盖** `venv/`、`config/`、`logs/`、`__pycache__/`、`.env` 目录
- 如果设置了环境变量 `ARBORE_OTA_DISABLED=1`，OTA 功能会被禁用
- 更新完成后服务会自动 `systemctl restart`，期间页面短暂不可访问（约 10~30 秒）

## 方式二：手动 scp 更新

适用于无法使用 OTA 的场景（如目标机无法访问公司服务器）。

### 更新后端

在**本机 PowerShell** 执行：

```powershell
scp "e:\Arbore\web-admin\backend\main.py" arbore@<目标机IP>:/home/arbore/console/web-admin/backend/
```

> 如果 arbore 用户无写入权限，先传到 `/tmp` 再 `sudo cp`：
> ```powershell
> scp "e:\Arbore\web-admin\backend\main.py" arbore@<目标机IP>:/tmp/main.py
> ```
> ```bash
> sudo cp /tmp/main.py /home/arbore/console/web-admin/backend/main.py
> sudo chown arbore:arbore /home/arbore/console/web-admin/backend/main.py
> ```

在**服务器**上重启：

```bash
sudo systemctl restart arbore-web-admin-api
```

### 更新前端

在**本机**构建并上传：

```powershell
cd e:\Arbore\web-admin\frontend
npm run build
scp -r dist arbore@<目标机IP>:/home/arbore/console/web-admin/frontend/
```

前端是静态文件，上传后无需重启，刷新浏览器（Ctrl+F5）即可。

### 通过公司服务器中转（目标机不能直连本机时）

1. 本机上传到公司服务器：

```powershell
scp "e:\Arbore\web-admin\backend\main.py" root@119.45.212.137:/tmp/web-admin-backend-main.py
```

2. 目标机从公司服务器下载：

```bash
scp root@119.45.212.137:/tmp/web-admin-backend-main.py /tmp/main.py
sudo cp /tmp/main.py /home/arbore/console/web-admin/backend/main.py
sudo chown arbore:arbore /home/arbore/console/web-admin/backend/main.py
sudo systemctl restart arbore-web-admin-api
```

## 方式三：更新 arbore-flow 镜像

修改 flow 相关代码后需重新构建 Docker 镜像（参见 `flow構建方法.md`）。

在**本机**构建并导出：

```powershell
docker build -t arbore-prod:001.00XX .
docker save arbore-prod:001.00XX -o arbore-prod.tar
```

上传到服务器并加载：

```powershell
scp arbore-prod.tar arbore@<目标机IP>:/home/arbore/console/
```

```bash
cd /home/arbore/console
docker load -i arbore-prod.tar
docker stop arbore-flow
docker rm arbore-flow
# 使用新镜像标签重新启动（参见 部署.md 第六步）
```

## 验证更新

| 检查项 | 方法 |
|--------|------|
| 后端版本 | 浏览器右上角版本号，或 `curl http://127.0.0.1:8002/api/v1/version` |
| 后端运行状态 | `sudo systemctl status arbore-web-admin-api` |
| 后端日志 | `sudo journalctl -u arbore-web-admin-api -f` |
| 前端是否生效 | 浏览器 Ctrl+F5 强制刷新 |
