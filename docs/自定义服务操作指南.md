# 自定义服务操作指南

**文档版本**: v1.1.0  
**最后更新**: 2025-01-XX  
**更新说明**: 添加卷挂载（volumes）配置功能支持

## 一、打包Docker镜像为tar文件

### 1. 构建Docker镜像

```bash
# 在服务代码目录下执行
docker build -t your-service-name:1.0.0 .
```

### 2. 导出为tar文件

```bash
# 导出镜像为tar文件
docker save -o your-service-name.tar your-service-name:1.0.0
```

**注意**：确保镜像名称和tag与后续部署时使用的名称一致。

## 二、添加服务

### 操作步骤

1. 登录Web管理控制台，进入"自定义服务管理"页面
2. 点击"添加服务"按钮
3. 填写以下信息：

| 字段 | 说明 | 限制 |
|------|------|------|
| **服务图标** | 选择服务图标 | 从预定义图标列表中选择 |
| **服务名称** | 服务的唯一标识 | • 只能包含小写字母、数字和连字符<br>• 不能为空<br>• 必须唯一（不能与现有服务重复）<br>• 示例：`my-api-service` |
| **描述信息** | 服务的描述说明 | • 可选，建议填写<br>• 用于说明服务的用途 |
| **内部端口** | 容器内部端口 | • 范围：7000-7999<br>• 必须唯一（不能与现有服务重复）<br>• 默认：7000 |
| **API端点** | 访问路径 | • 不能为空<br>• 不需要包含开头的斜杠<br>• 默认与服务名称一致<br>• 必须唯一（不能与现有服务重复）<br>• 示例：`api/my-service` |
| **Docker镜像** | tar文件 | • 必须上传<br>• 文件格式：`.tar`<br>• 最大大小：20GB |
| **环境变量** | 服务运行时环境变量 | • 可选配置<br>• 支持两种模式：表单模式和JSON模式<br>• 表单模式：逐行添加键值对<br>• JSON模式：直接输入JSON格式<br>• 环境变量会保存到 `config/{服务名称}/.env` 文件中<br>• 示例：`{"AUTH_ENABLED": "true", "SM4_KEY": "your_key"}` |
| **卷挂载** | 挂载宿主机目录到容器 | • 可选配置<br>• 支持两种模式：表单模式和JSON模式<br>• 表单模式：逐行添加挂载项（宿主机路径:容器路径，可选只读）<br>• JSON模式：直接输入JSON数组格式<br>• 格式：`["/host/path:/container/path", "/host/path2:/container/path2:ro"]`<br>• 示例：`["/path/to/huggingface/models:/models"]` |

4. 配置环境变量（可选）：
   - **表单模式**：点击"+ 添加变量"按钮，逐行添加环境变量的键值对
   - **JSON模式**：切换到"JSON模式"标签页，直接输入JSON格式的环境变量配置
   - 环境变量会在容器启动时自动加载到服务中

5. 配置卷挂载（可选）：
   - **表单模式**：点击"+ 添加挂载"按钮，逐行添加挂载项
     - 宿主机路径：宿主机上的目录路径（如：`/path/to/huggingface/models`）
     - 容器内路径：容器内的挂载点（如：`/models`）
     - 只读：勾选后挂载为只读模式（添加 `:ro` 后缀）
   - **JSON模式**：切换到"JSON模式"标签页，直接输入JSON数组格式
     - 格式：`["宿主机路径:容器路径", "宿主机路径2:容器路径2:ro"]`
     - 示例：`["/path/to/huggingface/models:/models"]` 或 `["/data:/app/data:ro"]`
   - 卷挂载会在容器启动时自动挂载到服务中

6. 点击"部署"按钮
7. 等待部署完成（通常需要1-2分钟）

### 字段验证规则

- **服务名称唯一性**：系统会检查是否已存在同名服务
- **端口唯一性**：系统会检查端口是否已被其他服务使用
- **端点唯一性**：系统会检查API端点是否已被其他服务使用
- **参数验证**：在上传文件之前会先验证参数，避免浪费时间和资源

## 三、修改服务

### 操作步骤

1. 在服务列表中找到要修改的服务
2. 点击"编辑"按钮
3. 修改以下信息（**注意**：修改服务时**不需要**重新上传tar文件）：

| 字段 | 是否可修改 | 说明 |
|------|-----------|------|
| **服务图标** | ✅ 可修改 | 可以更换图标 |
| **服务名称** | ❌ 不可修改 | 服务名称创建后不能修改 |
| **描述信息** | ✅ 可修改 | 可以更新描述 |
| **内部端口** | ✅ 可修改 | 可以修改端口（需满足唯一性要求） |
| **API端点** | ✅ 可修改 | 可以修改访问路径（需满足唯一性要求） |
| **Docker镜像** | ❌ 不可修改 | 修改服务时不支持重新上传镜像 |
| **环境变量** | ✅ 可修改 | 可以添加、修改或删除环境变量，支持表单模式和JSON模式 |
| **卷挂载** | ✅ 可修改 | 可以添加、修改或删除卷挂载，支持表单模式和JSON模式 |

4. 修改环境变量（如需要）：
   - 系统会自动加载现有的环境变量配置
   - 可以在表单模式或JSON模式中修改环境变量
   - 修改后的环境变量会覆盖原有配置

5. 修改卷挂载（如需要）：
   - 系统会自动加载现有的卷挂载配置
   - 可以在表单模式或JSON模式中修改卷挂载
   - 修改后的卷挂载会覆盖原有配置

6. 点击"保存"按钮
7. 系统会自动更新nginx配置并重启nginx

### 修改限制

- 修改端口时，新端口必须在7000-7999范围内，且不能与其他服务冲突
- 修改API端点时，新端点不能与其他服务冲突
- 如果需要更新Docker镜像，请先删除服务，然后重新添加

## 四、删除服务

### 操作步骤

1. 在服务列表中找到要删除的服务
2. 点击"删除"按钮
3. 确认删除操作
4. 系统会自动：
   - 停止并删除Docker容器
   - 从docker-compose.yml中删除服务配置
   - 从nginx配置中删除location块
   - 删除环境变量配置文件（`config/{服务名称}/.env`）
   - 重启nginx服务

**注意**：删除操作不可恢复，请谨慎操作。删除服务会同时删除所有相关配置，包括环境变量配置。

## 五、查询服务

### 操作步骤

1. 登录Web管理控制台，进入"自定义服务管理"页面
2. 服务列表会自动显示所有已部署的自定义服务
3. 列表显示的信息包括：
   - **图标**：服务图标
   - **服务名称**：服务标识
   - **描述**：服务描述
   - **端口**：容器内部端口
   - **API端点**：访问路径
   - **操作**：编辑/删除按钮

### 服务访问

部署成功后，可以通过以下方式访问服务：

```
http://服务器IP/API端点/
```

例如：如果API端点是 `api/my-service`，则访问地址为：
```
http://192.168.99.226/api/my-service/
```

## 六、常见问题

### 1. 部署失败怎么办？

- 检查服务名称、端口、端点是否唯一
- 检查tar文件是否正确（使用 `docker load -i your-service.tar` 测试）
- 查看后端日志：`sudo journalctl -u arbore-web-admin-api -f`

### 2. 服务无法访问？

- 检查容器是否运行：`docker ps | grep 服务名称`
- 检查nginx配置：`docker compose exec -T nginx nginx -t`
- 检查nginx日志：`docker compose logs nginx`

### 3. 如何更新服务镜像？

1. 删除现有服务
2. 重新打包新的tar文件
3. 重新添加服务

### 4. 端口冲突怎么办？

- 选择7000-7999范围内的其他端口
- 检查现有服务使用的端口：在服务列表中查看

### 5. 如何设置环境变量？

- **添加服务时**：在"环境变量"配置区域，选择表单模式或JSON模式进行配置
- **修改服务时**：点击"编辑"按钮，在环境变量配置区域修改现有配置
- **环境变量存储**：环境变量保存在 `config/{服务名称}/.env` 文件中
- **格式要求**：
  - 表单模式：变量名和变量值不能为空
  - JSON模式：必须是有效的JSON格式，例如：`{"KEY1": "value1", "KEY2": "value2"}`
- **生效方式**：环境变量会在容器启动时自动加载，修改后需要重启服务才能生效

### 6. 环境变量修改后不生效？

- 检查环境变量格式是否正确（JSON模式需要符合JSON语法）
- 确认已点击"保存"按钮保存配置
- 重启服务容器：`docker compose restart {服务名称}`
- 查看容器日志确认环境变量是否正确加载

### 7. 如何配置卷挂载？

- **添加服务时**：在"卷挂载"配置区域，选择表单模式或JSON模式进行配置
- **修改服务时**：点击"编辑"按钮，在卷挂载配置区域修改现有配置
- **格式要求**：
  - 表单模式：宿主机路径和容器路径不能为空
  - JSON模式：必须是有效的JSON数组格式，例如：`["/host/path:/container/path", "/host/path2:/container/path2:ro"]`
- **生效方式**：卷挂载会在容器启动时自动挂载，修改后需要重启服务才能生效
- **只读模式**：在表单模式中勾选"只读"选项，或在JSON模式中添加 `:ro` 后缀
- **示例**：挂载宿主机模型目录到容器
  - 宿主机路径：`/path/to/huggingface/models`
  - 容器内路径：`/models`
  - JSON格式：`["/path/to/huggingface/models:/models"]`

## 七、注意事项

1. **服务名称**：创建后不能修改，请谨慎命名
2. **端口范围**：只能在7000-7999范围内选择
3. **唯一性要求**：服务名称、端口、API端点都必须唯一
4. **文件大小**：tar文件最大支持20GB
5. **部署时间**：部署通常需要1-2分钟，请耐心等待
6. **删除操作**：删除服务会同时删除容器和配置，操作不可恢复
7. **环境变量**：
   - 环境变量配置是可选的，根据服务实际需求进行配置
   - 环境变量会保存到 `config/{服务名称}/.env` 文件中
   - 修改环境变量后，建议重启服务以确保新配置生效
   - 敏感信息（如密钥、密码）建议通过环境变量配置，不要硬编码在镜像中
   - JSON模式下，确保JSON格式正确，否则可能导致配置失败

8. **卷挂载**：
   - 卷挂载配置是可选的，用于将宿主机目录挂载到容器中
   - 卷挂载会直接写入 `docker-compose.yml` 文件的 `volumes` 配置中
   - 修改卷挂载后，建议重启服务以确保新配置生效
   - 挂载路径必须是绝对路径
   - 只读模式可以保护宿主机文件不被容器修改
   - JSON模式下，确保JSON数组格式正确，每个挂载项格式为 `"host_path:container_path"` 或 `"host_path:container_path:ro"`
   - 示例场景：挂载模型文件、数据目录、配置文件等到容器中
