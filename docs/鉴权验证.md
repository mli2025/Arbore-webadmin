# 鉴权/注册校验说明（桌面端联动）

## 目标

桌面端程序需要在启动前确认服务器的 Arbore 服务已完成“注册/授权”（License Registered）。本项目后端已内置许可证校验机制，通常**无需改代码**，只需要在新服务器上完成一次注册，并在桌面端调用接口进行检测即可。

> 重要：许可证绑定“宿主机”身份。更换服务器（宿主机 `machine-id` 变化）后必须重新注册。

## 桌面端需要做什么

桌面端只需要调用下述接口获取状态：

- `GET /api/v1/license`
  - 若返回 `registered: true`（或 `valid: true`），视为已注册，允许正常启动
  - 若为 `false`，可结合 `errorCode` 提示用户先在服务器上完成注册

如果后端通过 Nginx 反向代理暴露为 `/admin-api/`，则路径通常为：

- `GET /admin-api/api/v1/license`

## 现有后端接口清单

### 1) 查询注册状态

- **Method**：GET
- **Path**：`/api/v1/license`
- **说明**：返回是否已注册、错误码、宿主机指纹等信息
- **关键字段**：
  - `registered` / `valid`：是否有效
  - `errorCode`：无效原因（见下方）
  - `hostFingerprint`：宿主机展示指纹（形如 `host-xxxxxxxxxxxxxxxx`）

### 2) 获取宿主机指纹

- **Method**：GET
- **Path**：`/api/v1/license/hardware-fingerprint`
- **说明**：用于注册时展示/获取 `hostFingerprint`

### 3) 注册许可证

- **Method**：POST
- **Path**：`/api/v1/license/register`
- **Content-Type**：`multipart/form-data`（表单）
- **字段**：
  - `companyId`
  - `companyName`
  - `validationCode`（16位大写）

### 4) 校验当前许可证

- **Method**：POST
- **Path**：`/api/v1/license/validate`
- **说明**：返回 `valid`、`errorCode` 以及解析后的 license 信息

## 错误码（errorCode）

- `LICENSE_NOT_FOUND`：服务器尚未写入 `license.json`
- `LICENSE_INVALID`：校验码不匹配（companyId/hostFingerprint/versionMajor/secret 任一不一致）
- `HARDWARE_MISMATCH`：无法读取宿主机 `machine-id`（容器未正确挂载等）
- `VERSION_MISMATCH`：版本主段不一致（license 内存了版本且与当前不一致）
- `SECRET_NOT_CONFIGURED`：服务器端密钥不可用（通常不会出现；默认有内置密钥）

## 许可证绑定规则（必须了解）

### 1) 宿主机指纹来源

后端会读取宿主机 `machine-id`，并计算 SHA-256 形成稳定指纹，展示为：

- `hostFingerprint`：`host-` + 指纹前 16 位（示例：`host-a5b50f0617a557b8`）

容器化部署时，后端优先尝试读取：

1. 环境变量 `HOST_MACHINE_ID_FILE` 指向的文件
2. `/host/etc/machine-id`
3. `/etc/machine-id`

**因此 Docker 部署时需要将宿主机文件挂载到容器内**（例如挂载到 `/host/etc/machine-id`）。

### 2) validationCode 生成规则（16位）

生成方式与 `validation/validation-code-generator.js` 一致：

- `releaseVersion = version.split('.')[0].padStart(3, '0')`（仅取主段，补足3位）
- `data = "${companyId}-${hostId}-${releaseVersion}"`
  - `hostId` 使用接口返回的 `hostFingerprint`（形如 `host-xxxxxxxxxxxxxxxx`）
- `validationCode = HMAC_SHA256(secret, data).hex().substring(0, 16).toUpperCase()`

### 3) 密钥（secret）

默认固定密钥（后端与生成器一致）：

- `Arbore-License-Secret-Key-2024`

服务器端也支持通过环境变量覆盖：

- `ARBORE_LICENSE_SECRET` 或 `LICENSE_HMAC_SECRET`

**注意**：如果你们在服务器上配置了覆盖密钥，则生成验证码时必须使用同一密钥，否则会得到 `LICENSE_INVALID`。

## 服务器侧手工验证/注册命令（Linux Bash）

> 以下命令需要在服务器上执行。

### 1) 查看注册状态

```bash
curl -s http://127.0.0.1:8002/api/v1/license
```

### 2) 获取宿主机指纹

```bash
curl -s http://127.0.0.1:8002/api/v1/license/hardware-fingerprint
```

### 3) 注册（写入 license.json）

将下述参数替换为实际值：

```bash
curl -s -X POST http://127.0.0.1:8002/api/v1/license/register \
  -F "companyId=YOUR_COMPANY_ID" \
  -F "companyName=YOUR_COMPANY_NAME" \
  -F "validationCode=YOUR_16_CHAR_CODE"
```

## 桌面端集成建议（最小实现）

1. 配置服务器地址（例如 `https://<server>/admin-api` 或直连 `http://<server>:8002`）
2. 启动时调用 `GET /api/v1/license`
3. 若 `registered/valid` 为 `true`：继续启动
4. 否则：提示“服务器未注册/无效”，并可展示 `errorCode`
