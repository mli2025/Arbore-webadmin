# Arbore Web Admin 使用说明与功能说明

## 1. 软件概述

Arbore Web Admin 是面向本地部署环境的 Web 管理控制台，主要用于部署和管理本地零代码工作流引擎、RPA 机器人服务以及各类自定义服务。通过浏览器即可完成服务的安装、配置、升级与下线操作，无需在服务器上手工编辑配置文件或执行复杂命令，帮助运维与业务人员以更低门槛搭建本地自动化与 AI 服务体系。

## 2. 适用场景

- 本地或内网环境中，统一部署和管理 Arbore 系列标准服务（如工作流、函数服务、可视化服务、OCR 等）
- 在无公网依赖或高安全要求场景下，集中管理多个自定义业务服务和 AI 推理服务
- 通过本地零代码工作流和 RPA，将脚本、HTTP 接口、桌面操作等能力编排成可复用的自动化流程

## 3. 功能结构总览

- 标准服务管理：安装、卸载、修复、更新 Arbore 官方标准服务（含本地工作流、RPA 等）
- 自定义服务管理：通过 Docker 镜像 tar 文件一键部署自定义服务，支持环境变量与卷挂载配置
- 服务状态与访问：查看服务列表、运行状态、端口与访问路径，支持一键跳转至各服务的 Web 控制台
- 许可证与特性控制：根据许可证启用/禁用标准服务、自定义服务、RPA 等功能能力
- 运维辅助功能：集成日志查看、基本健康检查与常见问题排查指引（可按需扩展）

> 说明：以上为整体结构概览，可根据实际上线模块在后续章节中做细化调整。

## 4. 主要功能模块说明

### 4.1 标准服务管理

- 支持从标准服务清单中展示可用的 Arbore 官方服务（如工作流服务 flow、函数服务 func、可视化报表 quickchart、OCR 服务等）
- 对每个标准服务展示名称、描述、版本、安装状态（已安装 / 未安装 / 可更新）
- 提供“安装 / 卸载 / 修复 / 更新”等运维操作入口，底层通过维护 `docker-compose.yml` 和相关配置完成服务部署
- 可在界面中引导运维人员执行必要的 `docker compose` 命令，形成推荐的标准运维流程

> （如有更多标准服务类型或特殊流程，可在此处补充子模块说明）

### 4.2 自定义服务管理

- 通过图形化表单完成自定义服务的注册：
  - 选择服务图标、填写服务名称与描述
  - 配置容器内部端口与对外 API 端点，系统校验唯一性和端口范围
  - 上传打包好的 Docker 镜像 tar 文件
  - 使用表单模式或 JSON 模式配置环境变量与卷挂载
- 系统自动完成：
  - 将服务写入 `docker-compose.yml`
  - 生成对应的环境变量配置文件
  - 更新 Nginx 路由并重载相关配置
- 支持对已注册服务进行编辑（更新描述、端口、API 端点、环境变量、卷挂载等）以及安全删除（同时清理容器和配置）

### 4.3 服务状态与访问入口

- 在统一的服务列表中展示标准服务与自定义服务的汇总信息
- 提供基础状态信息（运行 / 停止 / 异常）、端口、API 端点等关键字段
- 支持一键跳转到对应服务的 Web 控制台或 API 根路径，方便调试与日常使用
- 通过刷新与状态聚合接口，保证在页面上能及时看到服务的最新运行情况

### 4.4 许可证与特性管理

- 读取并展示当前部署环境的许可证信息（如到期时间、授权级别、可用特性等）
- 根据许可证中定义的特性（如：标准服务、自定义服务、RPA 等）控制前端按钮启用状态和功能入口显示
- 为后续多档授权版本（基础版 / 标准版 / 高级版等）预留扩展空间

> （如有具体的授权策略、计费模式或更多特性位，可在此小节补充详细说明）

### 4.5 与本地工作流 / RPA 的协同

- 通过标准服务管理模块安装和升级本地工作流引擎与 RPA 机器人服务
- 为工作流和 RPA 服务提供统一的访问入口及基础健康状态显示
- 典型使用方式包括：
  - 在 Web Admin 中安装/升级工作流引擎服务
  - 通过工作流控制台配置零代码流程，将自定义服务作为节点接入
  - 通过 RPA 服务将桌面/网页操作与后端接口调用打通，形成端到端自动化流程

> （如工作流与 RPA 有更多具体产品形态或集成方式，可在此处细化子章节）

## 5. 典型使用流程示例

以下流程仅作为示例，可根据实际业务场景调整：

1. 运维人员在目标服务器上完成 Docker / docker compose 等基础环境准备  
2. 通过 Web Admin 部署 Arbore 标准服务：
   - 安装本地工作流服务（flow）
   - 安装 RPA 服务（如有）
3. 业务或运维人员登录工作流控制台，使用零代码界面编排流程：
   - 将已有 HTTP API、自定义服务、脚本任务等封装为节点
   - 配置触发条件（定时、事件、手动）
4. 通过“自定义服务管理”注册新的 AI 推理服务或业务服务，并暴露统一的 API 端点
5. 在工作流和 RPA 中引用这些自定义服务，构建端到端自动化方案
